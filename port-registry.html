Why Keep a Registry?

When running multiple services on one VPS, managing ports is critical. If you try to run two services on port 8080, the second one will crash.

This page acts as your "Air Traffic Control" to ensure no two services fight for the same space. Always check this list before assigning a port to a new Docker container.

How to Check Used Ports

If you are unsure if a port is free, run these commands on your VPS command line.

Method 1: Check Docker Containers

This shows exactly which ports Docker is holding open.

docker ps --format "table {{.Names}}\t{{.Ports}}"

Method 2: Check System Listeners

This shows absolutely everything listening on the server.

# The -tulpn flag shows TCP, UDP, Listening, Programs, Numeric addresses
sudo ss -tulpn
