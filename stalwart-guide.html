Prerequisites & Ports

Stalwart is a powerful mail server that handles SMTP, IMAP, and JMAP. Before installing, ensure your VPS firewall allows the following traffic:

  • Port 25 (TCP): SMTP (Receiving Mail). Note: Check if your VPS provider blocks outgoing port 25.
  • Port 587 (TCP): SMTP Submission (Sending Mail from clients).
  • Port 465 (TCP): SMTPS (Secure Submission).
  • Port 143 & 993 (TCP): IMAP (Reading Mail).
  • Port 4190 (TCP): ManageSieve (Filters).

Note: We will map the Web Admin Interface to port 8080 internally and expose it via Nginx Proxy Manager.

Part 1: Installation with Docker

We will set up Stalwart in its own directory with persistent storage.

Step 1: Create the Project Directory

# Create a dedicated directory for the mail server
mkdir -p /root/docker-deployment/mail
cd /root/docker-deployment/mail

Step 2: Create the `docker-compose.yml` File

nano docker-compose.yml

Paste the following configuration. This maps all necessary mail ports and connects the admin panel to our proxy network.

services:
  stalwart:
    image: stalwartlabs/stalwart:latest
    container_name: stalwart
    restart: unless-stopped
    ports:
      # SMTP (Receive Mail)
      - "25:25"
      
      # SMTP Submission (Send Mail - Client)
      - "587:587"
      
      # SMTPS (Secure Submission)
      - "465:465"
      
      # IMAP (Read Mail)
      - "143:143"
      - "993:993"
      
      # ManageSieve (Filters)
      - "4190:4190"
      
      # POP3 (Optional, usually not needed)
      - "110:110"
      - "995:995"

      # Web Admin Interface
      # We map this to host 8080.
      - "8080:8080"

    volumes:
      # Stores configuration and emails persistently
      - ./data:/opt/stalwart

    networks:
      - nginx-proxy

networks:
  nginx-proxy:
    external: true

Step 3: Launch Stalwart

Start the container in detached mode.

docker compose up -d

Part 2: Initial Login & Setup

On the very first run, Stalwart generates a random, secure password for the `admin` user. You need to view the logs to retrieve it.

1. Retrieve the Password

# View the container logs
docker compose logs

Look for a line that looks like this:

üîë Your administrator account is 'admin' with password 'geRMPA7oix'

Copy this password.

2. Access the Interface

Open your browser and navigate to http://YOUR_SERVER_IP:8080. Log in with:

  • Username: admin
  • Password: (The password you copied from the logs)

You will be greeted by the Setup Wizard. Follow the steps to configure your domain name.

Part 3: Changing the Admin Password

For security, you should change the generated password. However, Stalwart requires a restart for this specific change to take effect.

‚ö†Ô∏è Important: Changing the admin password is not instant. You must restart the container for the new password to work.

  1. In the Stalwart Web UI, go to Settings > Authentication.
  2. Update the password for the admin user and click Save.
  3. Restart the Container: You can do this easily via Portainer or the command line: 
    cd /root/docker-deployment/mail
docker compose restart

After the restart, you can log in with your new password.

Part 4: Securing with Nginx Proxy Manager

Accessing the admin panel via IP is insecure. Let's secure it with a domain and SSL.

  1. Point a Subdomain: In your domain registrar, create an 'A' record for a subdomain like mail.yourdomain.com pointing to your server's IP.
  2. Log in to Nginx Proxy Manager.
  3. Add a New Proxy Host:
    • Details Tab:
      • Domain Names: mail.yourdomain.com
      • Forward Hostname / IP: stalwart (Container name)
      • Forward Port: 8080
      • Websockets Support: Enabled (Required)
    • SSL Tab: Select "Request a new SSL Certificate" and turn on "Force SSL".
    • Save.

You can now securely manage your mail server at https://mail.yourdomain.com.

Part 5: Decommissioning

If you need to remove the mail server entirely:

  1. Remove the Proxy Host: In Nginx Proxy Manager, delete the host for `mail.yourdomain.com`.
  2. Stop and Remove Data: 
    cd /root/docker-deployment/mail
docker compose down
# ‚ö†Ô∏è WARNING: This deletes all emails and configuration!
rm -rf data
  3. Remove Directory: 
    cd ..
rm -rf mail
