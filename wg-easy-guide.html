Part 1: Setting Up WG-Easy with Docker

WG-Easy is a fantastic self-hosted WireGuard VPN server that includes a simple web interface for creating and managing users, making it perfect for our setup.

Step 1: Create the Project Directory

# Create a dedicated directory for the WG-Easy service
mkdir /root/docker-deployment/wg-easy

Step 2: Create the `docker-compose.yml` File

nano /root/docker-deployment/wg-easy/docker-compose.yml

Paste the following configuration. This file connects WG-Easy to the shared network, allowing the web UI to be secured. Be sure to edit the two required variables.

services:
  wg-easy:
    image: weejewel/wg-easy:latest
    container_name: wg-easy
    environment:
      # --- REQUIRED VARIABLES (EDIT THESE) ---

      # 1. Set the public IP address or a dedicated domain for the VPN connection.
      # This is used to generate correct client config files.
      - WG_HOST=YOUR_SERVER_IP

      # 2. Set the password you will use to log in to the web UI.
      # Change this to something strong and secret!
      - PASSWORD=YOUR_STRONG_PASSWORD

      # --- OPTIONAL VARIABLES (defaults are usually fine) ---

      # The port WireGuard listens on for VPN traffic.
      - WG_PORT=51820

      # The default DNS servers pushed to clients.
      # Cloudflare is a great default. See a list of alternatives below.
      - WG_DEFAULT_DNS=1.1.1.1,1.0.0.1
    volumes:
      # This maps the configuration directory to the host for persistence.
      - .:/etc/wireguard
    ports:
      - "51820:51820/udp" # WireGuard VPN Port
      - "51821:51821/tcp" # Web UI Port
    restart: unless-stopped
    cap_add:
      - NET_ADMIN
      - SYS_MODULE
    sysctls:
      - net.ipv4.conf.all.src_valid_mark=1
      - net.ipv4.ip_forward=1
    # Connect the container to our shared proxy network.
    networks:
      - nginx-proxy

# Declare that our shared network is external and pre-existing.
networks:
  nginx-proxy:
    external: true

Firewall Note: Before starting, ensure your VPS provider's firewall allows incoming traffic on port 51821/TCP (for the web UI) and port 51820/UDP (for the VPN connection).

Step 3: Launch WG-Easy

cd /root/docker-deployment/wg-easy
docker compose up -d

Part 2: Using the Web UI

  1. Access the UI by navigating to http://YOUR_SERVER_IP:51821 in your browser.
  2. Log in with the password you set in the `PASSWORD` variable.
  3. Click the **"+ New Client"** button to create your first VPN user.
  4. A new row will appear. You can click the client's name to rename it (e.g., `my-laptop`).
  5. To connect a device, use one of the icons:
    • QR Code Icon: Best for mobile phones. Scan it with the WireGuard app.
    • Download Icon: Best for computers. This saves a .conf file that you can import into your computer's WireGuard app.

Part 3: Customizing Client Configurations

While WG-Easy sets great defaults, you can manually edit the downloaded .conf file in a text editor for advanced customization.

Example .conf File

[Interface]
PrivateKey = [REDACTED - DO NOT CHANGE OR SHARE]
Address = 10.8.0.2/24
DNS = 1.1.1.1,1.0.0.1

[Peer]
PublicKey = [REDACTED - DO NOT CHANGE]
PresharedKey = [REDACTED - DO NOT CHANGE]
AllowedIPs = 0.0.0.0/0, ::/0
PersistentKeepalive = 0
Endpoint = YOUR_SERVER_IP:51820

What You Can (and Can't) Change

  • NEVER Change: The `PrivateKey`, `PublicKey`, and `PresharedKey` values. These are the cryptographic keys that secure your connection.

  • DNS: Change this to use a different DNS provider. This is the most common customization.

  • Endpoint: Change this if your server's IP changes. For a more professional setup, you can use a direct domain name (e.g., `wg.yourdomain.com:51820`). See the important note below.

Important: Setting the Correct VPN Endpoint

The Endpoint = ... line in your client config files is automatically generated from the WG_HOST variable in your docker-compose.yml.

Crucially, the domain you use for the Web UI (via Nginx Proxy Manager) CANNOT be used for WG_HOST. This is because NPM handles web traffic (TCP), while the VPN requires a direct connection (UDP).

What to put for the WG_HOST variable in your docker-compose.yml:

  • Option A (Simple): Use your server's direct IP address.
    - WG_HOST=YOUR_SERVER_IP
  • Option B (Recommended): Create a separate DNS 'A' record (e.g., wg.yourdomain.com) that points directly to your server's IP. Do not add this domain to Nginx Proxy Manager.
    - WG_HOST=wg.yourdomain.com

If you change this later, update your docker-compose.yml, run docker compose up -d from that directory, and re-download your client configs.

  • AllowedIPs: Change this for "split-tunneling". The default `0.0.0.0/0, ::/0` sends all traffic through the VPN. Changing it to `10.8.0.0/24` would only send traffic destined for your internal VPN network through the tunnel.

    • Recommended DNS Providers

    Here are some excellent public DNS providers you can use in the `DNS = ` line of your config file (or as the default in your `docker-compose.yml`).

    • General Purpose (Fast & Reliable):
      • Cloudflare: `1.1.1.1, 1.0.0.1`
      • Google: `8.8.8.8, 8.8.4.4`
    • Security-Focused (Blocks Malicious Domains):
      • Quad9: `9.9.9.9, 149.112.112.112`
      • Cloudflare Security: `1.1.1.2, 1.0.0.2`
    • Ad-Blocking:
      • AdGuard DNS: `94.140.14.14, 94.140.15.15`

    Part 4: Ongoing Management (Updating & Uninstalling)

    Updating WG-Easy

    To update to the latest version, follow the standard Docker Compose workflow:

    # Navigate to the WG-Easy directory
cd /root/docker-deployment/wg-easy

# Pull the latest image from the registry
docker compose pull

# Recreate the container with the new image
docker compose up -d

    Uninstalling WG-Easy

    To completely remove WG-Easy from your server:

    # 1. Navigate to the WG-Easy directory
cd /root/docker-deployment/wg-easy

# 2. Stop the container and remove its data volume
docker compose down -v

# 3. Go up one directory
cd ..

# 4. Be careful! This deletes the project folder permanently.
rm -rf wg-easy

    Part 5: Securing the UI with Nginx Proxy Manager

    It's highly recommended to put your WG-Easy admin panel behind a secure domain. The process is simple with the Nginx Proxy Manager you already have set up.

    1. Point a subdomain (e.g., `vpn.yourdomain.com`) to your server's IP.
    2. In Nginx Proxy Manager, add a new Proxy Host.
    3. Details Tab:
      • Domain Names: `vpn.yourdomain.com`
      • Forward Hostname / IP: `wg-easy` (Docker's internal DNS will resolve this).
      • Forward Port: `51821`
    4. SSL Tab: Request a new SSL certificate and turn on "Force SSL".
    5. Save and you're done! You can now access your panel securely at https://vpn.yourdomain.com.
