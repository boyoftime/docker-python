Part 1: Setting Up WG-Easy with Docker

WG-Easy is a fantastic self-hosted WireGuard VPN server. There are two common versions available. Please choose ONE of the options below.

Step 1: Create the Project Directory

Regardless of which version you choose, start by creating a folder:

# Create a dedicated directory
mkdir /root/docker-deployment/wg-easy
cd /root/docker-deployment/wg-easy

Step 2: Create the `docker-compose.yml` File

Create the file:

nano docker-compose.yml

Now, copy the code from ONE of the options below:

OPTION A: LATEST VERSION (v15+) - RECOMMENDED

This config uses the new SERVER_PUBLIC_URL variables. Data is stored locally in this folder.

services:
  wg-easy:
    # Uses the new official image repository
    image: ghcr.io/wg-easy/wg-easy:15
    container_name: wg-easy
    environment:
      # --- REQUIRED VARIABLES (New Format) ---
      # 1. Public IP or Domain
      - SERVER_PUBLIC_URL=https://YOUR_IP_OR_DOMAIN
      
      # 2. Web UI Password
      - ADMIN_PASSWORD=YOUR_STRONG_PASSWORD
      
      # --- CONFIGURATION ---
      - PORT=51821
      - WG_PORT=51820
      - DNS=1.1.1.1,1.0.0.1
      
    volumes:
      # Stores config files IN THIS DIRECTORY (as requested)
      - .:/etc/wireguard
      # Required for v15+ networking
      - /lib/modules:/lib/modules:ro

    ports:
      - "51820:51820/udp" # VPN Port
      - "51821:51821/tcp" # Web UI Port

    restart: unless-stopped
    cap_add:
      - NET_ADMIN
      - SYS_MODULE
    sysctls:
      - net.ipv4.conf.all.src_valid_mark=1
      - net.ipv4.ip_forward=1
      - net.ipv6.conf.all.disable_ipv6=0
      - net.ipv6.conf.all.forwarding=1
      - net.ipv6.conf.default.forwarding=1

    networks:
      - nginx-proxy

networks:
  nginx-proxy:
    external: true
OPTION B: LEGACY VERSION (WeeJewel)

Use this if you prefer the legacy setup (old variable names).

services:
  wg-easy:
    # Uses the original legacy image
    image: weejewel/wg-easy:latest
    container_name: wg-easy
    environment:
      # --- REQUIRED VARIABLES ---
      - WG_HOST=YOUR_SERVER_IP
      - PASSWORD=YOUR_STRONG_PASSWORD
      
      # --- CONFIGURATION ---
      - WG_PORT=51820
      - WG_DEFAULT_DNS=1.1.1.1,1.0.0.1

    volumes:
      # Stores config files in the same directory as this docker-compose file
      - .:/etc/wireguard

    ports:
      - "51820:51820/udp"
      - "51821:51821/tcp"
    
    restart: unless-stopped
    cap_add:
      - NET_ADMIN
      - SYS_MODULE
    sysctls:
      - net.ipv4.conf.all.src_valid_mark=1
      - net.ipv4.ip_forward=1

    networks:
      - nginx-proxy

networks:
  nginx-proxy:
    external: true

Step 3: Launch WG-Easy

‚ö†Ô∏è If you get a "Migration" Error (e.g. "invalid Configuration"):

If you see an error in the logs about migration, it means you have an old incompatible configuration file in this folder.

To fix this, you must delete the old file to start fresh:

rm /root/docker-deployment/wg-easy/config.json
rm /root/docker-deployment/wg-easy/wg0.json

Start the server normally:

cd /root/docker-deployment/wg-easy
docker compose up -d

Part 2: Using the Web UI

  1. Access the UI by navigating to http://YOUR_SERVER_IP:51821 in your browser.
  2. Log in with the password you set in the config file.
  3. Click the **"+ New Client"** button to create your first VPN user.
  4. To connect a device, use one of the icons:
    • QR Code Icon: Best for mobile phones. Scan it with the WireGuard app.
    • Download Icon: Best for computers. This saves a .conf file.

Part 3: Customizing Client Configurations

You can manually edit the downloaded .conf file in a text editor for advanced customization.

Important: Setting the Correct VPN Endpoint

The Endpoint = ... line in your client config files is automatically generated from the SERVER_PUBLIC_URL (or WG_HOST) variable.

The domain you use for the Web UI (via Nginx Proxy Manager) CANNOT be used for the VPN endpoint unless you open UDP ports on that specific domain.

Your Public URL variable should be:

  • Option 1: Your server's IP address (e.g., https://123.45.67.89).
  • Option 2: A separate DNS 'A' record (e.g., https://wg.yourdomain.com) pointing directly to the IP.

Part 4: Ongoing Management

Updating

cd /root/docker-deployment/wg-easy
docker compose pull
docker compose up -d

Uninstalling

cd /root/docker-deployment/wg-easy
docker compose down
cd ..
rm -rf wg-easy

Part 5: Securing the UI with Nginx Proxy Manager

Secure the admin panel (TCP port 51821) using SSL.

  1. Point a subdomain (e.g., `vpn.yourdomain.com`) to your server's IP.
  2. In Nginx Proxy Manager, add a new Proxy Host.
  3. Details Tab:
    • Domain Names: `vpn.yourdomain.com`
    • Forward Hostname / IP: `wg-easy`
    • Forward Port: `51821`
  4. SSL Tab: Request a new SSL certificate and turn on "Force SSL".
