Why Secure the Admin Panel?

Accessing your Nginx Proxy Manager (NPM) admin panel via an IP address like http://101.102.103.25:81 is fine for the initial setup. However, for long-term use, it has two major drawbacks:

  • It's Insecure: Your login credentials are sent over the internet in plain text (HTTP).
  • It's Inconvenient: It's hard to remember an IP address and port number.

By using NPM to manage itself, we can solve both problems, giving you a secure, easy-to-remember HTTPS domain for your most important control panel.

Prerequisites

  • You must have completed the initial setup of Nginx Proxy Manager from the main guide.
  • You must have a domain name ready to use for the admin panel.

The Setup Process

The process involves telling NPM to forward traffic from a public domain to its own admin panel, which is running on the server itself.

  1. Point a Subdomain: In your domain registrar (like Namecheap, GoDaddy, etc.), create a new 'A' record for a subdomain. For example, use proxy.yourdomain.com and point it to your server's public IP address.
  2. Log in to Nginx Proxy Manager: For the last time, go to http://YOUR_SERVER_IP:81 to access the admin panel.
  3. Add a New Proxy Host: Navigate to Hosts → Proxy Hosts and click "Add Proxy Host".
  4. Fill in the Details Tab:
    • Domain Names: Enter the subdomain you just created, e.g., proxy.yourdomain.com.
    • Scheme: Leave this as http.
    • Forward Hostname / IP: Enter 127.0.0.1. This special IP address means "this server itself" (also known as localhost).
    • Forward Port: Enter 81. This is the port for the admin panel.
  5. Configure the SSL Tab:
    • SSL Certificate: Select "Request a new SSL Certificate".
    • Turn on "Force SSL".
    • Agree to the Terms of Service and click Save.

‚ú® After about a minute, the process will complete. You can now access your secure, professional Nginx Proxy Manager dashboard at https://proxy.yourdomain.com. You should bookmark this new URL and use it for all future access.

How does this work? It might seem strange to proxy traffic to `127.0.0.1` (localhost). This is a clever technique that tells Nginx Proxy Manager: "When you receive a request for `proxy.yourdomain.com`, don't send it to another container. Instead, forward it internally to port 81 on the host server itself." Since the admin panel is exposed on port 81, this connects the public domain to the private admin service securely.
