Handling URL Redirection with SSL Support in Nginx Proxy Manager

Why Not Use CNAMEs or DNS Forwarding?

When you want www.kingscapitaltz.com to go to kingscapitaltz.com, beginners often make two common mistakes:

Mistake 1: Relying on CNAMEs

A CNAME Record only tells the DNS system: "If someone asks for www, send them to the same IP address as the main domain."

It does not change the URL in the browser bar. The user will still see www.kingscapitaltz.com in their address bar, which splits your SEO ranking and analytics between two different sites.

Mistake 2: Using Registrar "URL Redirects" (without SSL)

Most registrars (like Namecheap) offer a "URL Redirect" record type. While this works for HTTP, it often breaks **HTTPS**.

If a user types https://www..., the registrar's redirect server (which doesn't have your SSL certificate) will fail to handshake, and the user will see a scary "Your connection is not private" warning before the redirect even happens.

The Solution: "Redirection Hosts"

The professional way to handle this is to point both the @ (root) and www records to your server, and let Nginx Proxy Manager handle the logic.

By doing this on your own server:

  • You can generate **SSL Certificates** for both `www` and `non-www`.
  • The redirect happens securely over HTTPS (no warning screens).
  • You can preserve the path (e.g., `www.site.com/login` redirects correctly to `site.com/login`).

Step 1: The DNS Setup

Before configuring the proxy, ensure your DNS is pointing the traffic to your server.

  1. Log into your Domain Registrar (Namecheap).
  2. Delete any existing CNAME record for `www`.
  3. Add a new **A Record**:
    • Host: www
    • Value: YOUR_SERVER_IP (e.g., 172.112.208.15)

Wait! DNS changes can take a few minutes to propagate. If Nginx Proxy Manager gives an error when saving in the next step, wait 5 minutes and try again.

Step 2: Nginx Proxy Manager Setup

Now we tell your server to catch the `www` traffic and bounce it to the main domain.

1. Open Redirection Hosts

In Nginx Proxy Manager, click on Hosts in the top menu, then select Redirection Hosts.

Click Add Redirection Host.

2. The Details Tab

Fill out the form exactly as follows:

  • Domain Names: www.kingscapitaltz.com (Press Enter)
  • Scheme: https
  • Forward Domain: kingscapitaltz.com
  • Forward Port: 443 (Standard HTTPS port)
  • HTTP Code: 301 Moved permanently (Best for SEO)
  • Preserve Path: ENABLED (Switch On)
  • Block Common Exploits: ENABLED

3. The SSL Tab

We must generate a certificate for the `www` subdomain so the redirect is secure.

  • SSL Certificate: Select "Request a new SSL Certificate".
  • Force SSL: ENABLED
  • Email Address: Ensure your email is present.
  • Agree to Terms: ENABLED

Click Save.

Verification

To test this properly, open a "Private" or "Incognito" window in your browser (to avoid caching issues).

  1. Type http://www.kingscapitaltz.com -> Should redirect to https://kingscapitaltz.com
  2. Type https://www.kingscapitaltz.com -> Should redirect to https://kingscapitaltz.com
  3. Type https://www.kingscapitaltz.com/some-page -> Should redirect to https://kingscapitaltz.com/some-page
