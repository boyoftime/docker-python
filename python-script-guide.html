The Golden Rules of Background Scripts

Running a Python script in a Docker container is different from running it on your laptop. Because there is no screen and no keyboard attached to the container, you must follow these rules:

❌ WHAT YOU CANNOT DO

  • NO input(): You cannot ask the user to type anything. The script will crash immediately.
  • NO GUIs: You cannot use tkinter, pygame, or open web browser windows.
  • NO Interactive Menus: You cannot have "Press 1 for settings, Press 2 to run".

✅ WHAT YOU CAN DO

  • Hardcode Settings: You must write your API Keys, passwords, and settings directly into the top of the script variables.
  • Infinite Loops: Use while True: to keep the bot running forever.
  • Print to Logs: Use print() to show status updates in the Docker logs.

Step 1: Preparing the Project

Create a clean directory for your bot.

mkdir -p /root/docker-deployment/my-bot
cd /root/docker-deployment/my-bot

Create the python file:

nano bot.py

Step 2: Choose Your Script

Below are two sample scripts you can use to test your setup. Choose one, copy it into bot.py, and edit the configuration variables at the top.

Option A: The Telegram "Heartbeat" Bot

This script sends a message to your Telegram every hour to let you know your server is alive.

import requests
import time
from datetime import datetime

# ==========================================
# ⚙️ CONFIGURATION (EDIT THIS SECTION)
# ==========================================
TELEGRAM_BOT_TOKEN = "YOUR_BOT_TOKEN_HERE"
TELEGRAM_CHAT_ID = "YOUR_CHAT_ID_HERE"
MESSAGE_TEXT = "🤖 Beep Boop! Your server is running smoothly."
CHECK_INTERVAL_SECONDS = 3600  # Send every 1 hour
# ==========================================

def send_telegram(message):
    url = f"https://api.telegram.org/bot{TELEGRAM_BOT_TOKEN}/sendMessage"
    payload = {"chat_id": TELEGRAM_CHAT_ID, "text": message}
    try:
        response = requests.post(url, json=payload)
        if response.status_code == 200:
            print(f"[{datetime.now()}] ✅ Message sent successfully.")
        else:
            print(f"[{datetime.now()}] ❌ Failed to send: {response.text}")
    except Exception as e:
        print(f"[{datetime.now()}] ❌ Connection Error: {e}")

print("🚀 Bot started! Waiting to send first message...")

while True:
    send_telegram(MESSAGE_TEXT)
    print(f"Sleeping for {CHECK_INTERVAL_SECONDS} seconds...")
    time.sleep(CHECK_INTERVAL_SECONDS)

Option B: Bitcoin Price Tracker

This script checks the price of Bitcoin every 60 seconds and logs it to the console.

import requests
import time
from datetime import datetime

# ==========================================
# ⚙️ CONFIGURATION (EDIT THIS SECTION)
# ==========================================
CURRENCY = "bitcoin"
VS_CURRENCY = "usd"
CHECK_INTERVAL_SECONDS = 60
# ==========================================

print(f"🚀 Tracking {CURRENCY} price every {CHECK_INTERVAL_SECONDS} seconds...")

while True:
    try:
        url = f"https://api.coingecko.com/api/v3/simple/price?ids={CURRENCY}&vs_currencies={VS_CURRENCY}"
        response = requests.get(url)
        data = response.json()
        
        price = data[CURRENCY][VS_CURRENCY]
        
        # This print statement will show up in your Docker Logs/Portainer
        print(f"[{datetime.now().strftime('%H:%M:%S')}] 💰 Current Price: ${price}")
        
    except Exception as e:
        print(f"❌ Error fetching price: {e}")

    time.sleep(CHECK_INTERVAL_SECONDS)

Step 3: Dependencies & Dockerfile

1. Create requirements.txt

Tell Docker which libraries to install (e.g., `requests`).

nano requirements.txt

requests
certifi

2. Create Dockerfile

This tells Docker how to run Python. The -u flag is critical—it ensures logs show up instantly.

nano Dockerfile

FROM python:3.12-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY bot.py .

# The -u flag (unbuffered) lets you see print() statements instantly
CMD ["python", "-u", "bot.py"]

3. Create docker-compose.yml

nano docker-compose.yml

services:
  my-bot:
    build: .
    container_name: python_bot_1
    restart: unless-stopped
    volumes:
      # Allows you to edit bot.py on the server and restart to apply changes
      - ./bot.py:/app/bot.py

Step 4: Running & Monitoring

Start the Bot

cd /root/docker-deployment/my-bot
docker compose up -d --build

View the Logs (Real-time)

You will see your print() statements appear here.

docker compose logs -f

How to Update Settings

  1. Edit the script: nano bot.py
  2. Change your variables in the "CONFIGURATION" section.
  3. Restart the container: docker compose restart

Step 5: How to Uninstall

If you are done with the bot and want to remove it completely:

# 1. Stop and remove the container
cd /root/docker-deployment/my-bot
docker compose down

# 2. Delete the folder (PERMANENT)
cd ..
rm -rf my-bot
