Prerequisites & Ports

Before installing, you must ensure your VPS allows traffic on email ports. Check your VPS provider's firewall settings.

‚ö†Ô∏è Important Port Information:

  • Port 25 (SMTP): Required to receive mail. Many residential ISPs and some VPS providers block this by default. You may need to ask support to unblock it.
  • Port 443 (HTTPS): Stalwart usually wants this, but Nginx Proxy Manager is already using it. We will NOT map port 443 in this installation to avoid conflicts.

Part 1: Installation with Docker

Step 1: Create the Project Directory

Create a dedicated folder for the mail server files.

# Create the directory
mkdir /root/docker-deployment/mail
cd /root/docker-deployment/mail

Step 2: Create the `docker-compose.yml` File

Create the file:

nano docker-compose.yml

Paste the configuration below. IMPORTANT: Change 'password' to your own strong password.

services:
  stalwart:
    image: stalwartlabs/stalwart:latest
    container_name: stalwart
    restart: unless-stopped
    environment:
      # FORCE the admin password.
      # CHANGE 'password' TO YOUR OWN STRONG PASSWORD!
      - STALWART_AUTHENTICATION__FALLBACK_ADMIN__SECRET="your_password_here"
    ports:
      # SMTP (Receive Mail)
      - "25:25"
      
      # SMTP Submission (Send Mail - Client)
      - "587:587"
      
      # SMTPS (Secure Submission)
      - "465:465"
      
      # IMAP (Read Mail)
      - "143:143"
      - "993:993"
      
      # ManageSieve (Filters)
      - "4190:4190"
      
      # POP3 (Optional, usually not needed)
      - "110:110"
      - "995:995"

      # Web Admin Interface
      # We map this to host 8080.
      - "8080:8080"

    volumes:
      # Stores configuration and emails
      - ./data:/opt/stalwart

    networks:
      - nginx-proxy

networks:
  nginx-proxy:
    external: true

Step 3: Launch the Server

# Start the container
docker compose up -d

Part 2: Initial Login

Unlike the default installation, you do not need to check logs. You defined your password in the configuration file above.

URL: http://YOUR_SERVER_IP:8080

Username: admin

Password: The value you set for STALWART_AUTHENTICATION__FALLBACK_ADMIN__SECRET (e.g., your_password).

Part 3: Setup & Creating a Real User

Once logged in, follow the setup wizard.

Next Steps inside the UI:

  1. Go to Settings > Server > Network to set your hostname.
  2. Go to Management > Directory > Domains to add your domain (e.g., kingscapitaltz.com).
  3. The interface will then show you the exact DNS Records (MX, SPF, DKIM) you need to add to your Domain Registrar.
  4. CRITICAL: Create a Real Admin Account
    Even though we set the password in the file, it is best practice to create a database-backed user for day-to-day management.
    • Go to Management > Directory > Accounts.
    • Click Create Account.
    • Create a new user (e.g., superuser) with a strong password.
    • Important: Enable the "Superuser" or "Administrator" role for this user.
    • Log out and log back in with your new user.

Part 4: Securing the Web Admin with Nginx Proxy Manager

Accessing the admin panel via IP is not recommended for production. Let's secure it with a domain name and SSL.

  1. Point a Subdomain: In your domain registrar, create a new 'A' record (e.g., mail.yourdomain.com) pointing to your server's IP.
  2. Log in to Nginx Proxy Manager.
  3. Add a New Proxy Host:
    • Details Tab:
      • Domain Names: mail.yourdomain.com
      • Scheme: http
      • Forward Hostname / IP: stalwart (This matches the container name defined in docker-compose).
      • Forward Port: 8080 (The internal web port).
      • Turn on "Websockets Support" (Recommended for the live admin UI).
    • SSL Tab: Select "Request a new SSL Certificate" and turn on "Force SSL".
    • Save.

You can now access your secure mail server admin panel at https://mail.yourdomain.com.
