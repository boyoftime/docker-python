The "Hybrid" Philosophy: CLI for Control, GUI for Visibility

This guide will help you set up Portainer, a powerful web GUI for Docker. We will follow a common professional workflow that combines the strengths of the command line (CLI) and a graphical interface (GUI):

  • Deploy & Update with the CLI: You will continue to manage your project's lifecycle (deploying, updating, and rebuilding) from the command line using docker compose. This keeps your project's files as the "single source of truth" and is fast and reliable.
  • Monitor & Manage with Portainer: You will use the Portainer dashboard for all day-to-day monitoring and management tasks: checking logs, viewing resource usage (CPU/RAM), and performing quick actions like restarting a single container.

Part 1: Installing Portainer

Step 1: Create the Project Directory

Just like our other services, we will give Portainer its own organized folder.

# Create a dedicated directory for the Portainer service
mkdir /root/docker-deployment/portainer

Step 2: Create the `docker-compose.yml` for Portainer

Create and edit the compose file for Portainer.

nano /root/docker-deployment/portainer/docker-compose.yml

Paste the following configuration into the file. This version connects Portainer to the shared proxy network, which is essential for securing it with a domain name.

services:
  portainer:
    image: portainer/portainer-ce:latest
    container_name: portainer
    restart: unless-stopped
    ports:
      # Exposes Portainer's UI on port 9443 (HTTPS)
      - "9443:9443"
      # This port is also needed for the Portainer agent
      - "9000:9000" 
    volumes:
      # This is the "special telephone line" that lets Portainer control Docker
      - /var/run/docker.sock:/var/run/docker.sock
      # This persists Portainer's own data (users, settings, etc.)
      - ./portainer_data:/data
    # Connect this container to the shared proxy network.
    networks:
      - nginx-proxy

# This section tells Docker that 'nginx-proxy' is a pre-existing network
# and should not be created by this file.
networks:
  nginx-proxy:
    external: true

Step 3: Launch Portainer

If you are updating from the old configuration, running this command will safely recreate the container with the new network setting without losing any data.

# Navigate to the portainer directory
cd /root/docker-deployment/portainer

# Start the service
docker compose up -d

Part 2: First-Time Setup

  1. Open your browser and go to https://YOUR_SERVER_IP:9443.
  2. Your browser will show a security warning because the certificate is self-signed. This is normal. Click "Advanced" and "Proceed to..." to continue.
  3. Portainer will ask you to create a new `admin` user and a strong password. Do this now.
  4. On the next screen, it will ask which Docker environment you want to manage. Choose **"Get Started"** to manage the local Docker environment that Portainer is running in.

You will now be on the main Portainer dashboard, where you can see a summary of your Docker environment.

Part 3: The Hybrid Workflow in Action

This section explains how to use the CLI and Portainer together for a smooth and powerful workflow.

Understanding "Limited Control"

When you navigate to the Stacks menu in Portainer, you will see all your projects (`proxy-manager`, `flask_app_1`, etc.) listed, but they will be marked as **"Limited"**. This is normal and expected for our workflow.

It simply means that Portainer sees the stack is running, but it knows the "source of truth" is the `docker-compose.yml` file on your server, not a configuration stored in Portainer's database. This is perfect for our use case, as it still gives us full control over the individual containers.

Deploying and Updating with the CLI

This part of your workflow does not change. When you want to deploy a new app or update an existing one with new code, you will use the command line.

# Example for updating the first app
# 1. Upload your new code files with an SFTP client

# 2. SSH into your server and navigate to the app's directory
cd /root/docker-deployment/apps/flask_app_1

# 3. Run the build and deploy command
docker compose up --build -d

Monitoring and Managing Containers in Portainer

Once your application is running, Portainer becomes your "cockpit" for monitoring.

  1. From the Portainer home screen, click on the **"local"** environment.
  2. In the left menu, click on **Containers**. You will now see a list of all your running containers.
  3. Click on any container name (e.g., `flask_app_one_app`) to see its details.

From the Container details page, you can perform many useful actions:

  • Logs: View the real-time log output of your application. This is essential for debugging.

  • Inspect: See all the detailed Docker configuration for the container.

  • Stats: View a live graph of the CPU and Memory usage of your container.

  • Console: Open a command-line shell directly inside the running container for advanced troubleshooting.

  • Restart, Stop, Kill, Remove: The action buttons at the top allow you to control the container's state. `Restart` is particularly useful for a quick reboot without a full redeployment.

Part 4: Securing Portainer with a Domain and SSL (Optional)

Accessing Portainer via an IP address is fine, but it's more professional and secure to use a domain name with a valid SSL certificate. Since you already have Nginx Proxy Manager running, this is very easy.

  1. Point a Subdomain: In your domain registrar, create a new 'A' record for a subdomain like docker.yourdomain.com and point it to your server's IP address.
  2. Log in to Nginx Proxy Manager: Go to http://YOUR_SERVER_IP:81.
  3. Add a New Proxy Host:
    • Details Tab:
      • Domain Names: docker.yourdomain.com
      • Scheme: https
      • Forward Hostname / IP: portainer (Docker's internal DNS will resolve this to the Portainer container because it's on the same network).
      • Forward Port: 9443 (The secure port for Portainer's UI).
      • Turn on "Websockets Support", as Portainer uses it for live stats and logs.
    • SSL Tab: Select "Request a new SSL Certificate" and turn on "Force SSL".
    • Save.

After a minute, you can access your secure, professional Portainer dashboard at https://docker.yourdomain.com without any browser warnings.
