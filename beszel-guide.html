Setting Up Beszel System Monitoring

A Lightweight, Modern Dashboard for CPU, RAM, Disk, and Docker Stats

Why Beszel?

Unlike heavy monitoring tools (like Prometheus/Grafana), Beszel is incredibly lightweight. It gives you exactly what you need:

  • Historical data (CPU/RAM usage over time).
  • Specific Docker container usage (Find out which app is eating your RAM).
  • Disk space alerts.
  • A beautiful, modern interface.

Part 1: Installation with Docker

Beszel has two parts: The Hub (Dashboard) and the Agent (Data Collector). We will run both.

Step 1: Create the Project Directory

mkdir -p /root/docker-deployment/monitoring
cd /root/docker-deployment/monitoring

Step 2: Create the `docker-compose.yml` File

nano docker-compose.yml

Paste the following configuration. We use port 8094 for the dashboard.

services:
  # The Dashboard (UI)
  beszel-hub:
    image: henrygd/beszel:latest
    container_name: beszel-hub
    restart: unless-stopped
    ports:
      - "8094:8090"
    volumes:
      - ./beszel_data:/beszel_data
    networks:
      - nginx-proxy

  # The Agent (Reads the stats)
  beszel-agent:
    image: henrygd/beszel-agent:latest
    container_name: beszel-agent
    restart: unless-stopped
    network_mode: host # Required to read system stats accurately
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro # To read Docker stats
    environment:
      - PORT=45876
      # We will add the KEY here in the next step
      - KEY=
      # Optional: Monitor specific disk
      - FILESYSTEM=/dev/sda1

networks:
  nginx-proxy:
    external: true

Step 3: Initial Launch (Hub Only)

We start the Hub first to generate an encryption key.

docker compose up -d beszel-hub

Part 2: Configuring the Agent Key

The Agent needs a secure key to talk to the Hub. This requires a quick setup in the browser.

  1. Open Firewall: Ensure Port 45876 (TCP) is allowed in your VPS firewall (for the Agent).
  2. Access the Hub: Go to http://YOUR_SERVER_IP:8094.
  3. Create Admin Account: Follow the prompt to create your login.
  4. Add System:
    • Click "+ Add System".
    • Name: `My VPS`
    • Host / IP: Enter your server's Public IP Address.
    • Port: `45876`
    • Public Key: Click the Copy icon üìã next to the key starting with `ssh-ed25519`.
    • DO NOT click "Add system" yet!

Update Docker Compose

Go back to your terminal and edit the compose file to add the key.

nano docker-compose.yml

Find the beszel-agent section and paste your key:

    environment:
      - PORT=45876
      - KEY=ssh-ed25519 AAAAC3Nz... (Paste your exact key here)

Save and exit.

Start the Agent

docker compose up -d

Now, go back to your browser and click the "Add system" button. It should turn green and show your stats!

Part 3: Securing with Nginx Proxy Manager

Access the dashboard securely via a domain.

  1. Point a Subdomain: Create an 'A' record for `monitor.yourdomain.com` pointing to your server IP.
  2. Log in to Nginx Proxy Manager.
  3. Add Proxy Host:
    • Domain: `monitor.yourdomain.com`
    • Forward Hostname: `beszel-hub`
    • Forward Port: `8090` (Internal container port)
    • SSL: Request Certificate & Force SSL.

Part 4: Decommissioning

To remove the monitoring system:

# Remove Proxy Host in NPM first.

# Stop and remove containers and data
cd /root/docker-deployment/monitoring
docker compose down -v

# Remove directory
cd ..
rm -rf monitoring
