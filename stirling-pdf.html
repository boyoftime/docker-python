Why Stirling-PDF?

Stirling-PDF allows you to perform almost any operation on PDF files directly from your browser. Stop uploading your private documents to random "Free Online PDF Merger" websites.

Features Include: Splitting, Merging, Rotating, converting PDFs to Images (and vice versa), Password protection, OCR (Optical Character Recognition), and much more.

Choosing Your Version

The official Stirling-PDF image comes in three flavors. For most users, latest is the best choice.

Version Tag Description Best For
latest Standard. Includes all standard PDF features. Most users. Balanced features & size.
latest-fat Full. Everything + extra tools & conversion formats. Power users who need every possible format. Large download.
latest-ultra-lite Lite. Core features only. Low-end VPS or Raspberry Pi. Fastest startup.

Part 1: Standard Installation (Recommended)

We will set up the Standard version with full persistent data.

Step 1: Create the Project Directory

# Create a directory for the service
mkdir -p /root/docker-deployment/stirling-pdf
cd /root/docker-deployment/stirling-pdf

Step 2: Create the `docker-compose.yml` File

nano docker-compose.yml

Paste the following configuration.

services:
  stirling-pdf:
    image: stirlingtools/stirling-pdf:latest
    container_name: stirling-pdf
    restart: unless-stopped
    ports:
      # Maps host port 8085 to container port 8080
      - '8085:8080'
    volumes:
      - ./stirling-data/tessdata:/usr/share/tessdata  # OCR language files
      - ./stirling-data/configs:/configs               # Settings & database
      - ./stirling-data/logs:/logs                     # Application logs
      - ./stirling-data/pipeline:/pipeline             # Automation configs
    environment:
      - SECURITY_ENABLELOGIN=true  # Enable login (Recommended)
      - LANGS=en_US
    networks:
      - nginx-proxy

# Connect to your existing shared network
networks:
  nginx-proxy:
    external: true

Step 3: Fix Permissions and Launch

Since we are using volumes, we must ensure the folders exist with the correct permissions before starting.

# Create folder and set permissions
mkdir stirling-data
chmod -R 755 stirling-data

# Start the service
docker compose up -d

Default Credentials:
Username: admin
Password: stirling
(Please change these in the settings immediately)

Part 2: Securing with Nginx Proxy Manager

To access this securely via a domain name (e.g., `pdf.yourdomain.com`).

  1. Point a Subdomain: Create an 'A' record for your subdomain pointing to your server's IP.
  2. Log in to Nginx Proxy Manager.
  3. Add a New Proxy Host:
    • Domain Names: pdf.yourdomain.com
    • Forward Hostname / IP: stirling-pdf (Matches container name).
    • Forward Port: 8080 (Internal port).
    • SSL Tab: Request a new SSL Certificate and Force SSL.

Part 3: Advanced Split Deployment

For high-traffic setups, you can scale the Frontend and Backend independently using the V2.0+ architecture. Only use this if you know you need it.

Split `docker-compose.yml`

services:
  # Backend - PDF processing engine
  stirling-backend:
    image: stirlingtools/stirling-pdf:latest
    container_name: stirling-backend
    volumes:
      - ./stirling-data/tessdata:/usr/share/tessdata
      - ./stirling-data/configs:/configs
    environment:
      - MODE=BACKEND
    restart: unless-stopped
    networks:
      - nginx-proxy

  # Frontend - Web interface
  stirling-frontend:
    image: stirlingtools/stirling-pdf:latest
    container_name: stirling-frontend
    ports:
      - '8085:8080'
    environment:
      - MODE=FRONTEND
      # Internal DNS name of the backend service
      - VITE_API_BASE_URL=http://stirling-backend:8080
    depends_on:
      - stirling-backend
    restart: unless-stopped
    networks:
      - nginx-proxy

networks:
  nginx-proxy:
    external: true

Nginx Proxy Manager Note: If using Split Deployment, your Proxy Host must point to Hostname: stirling-frontend (NOT stirling-pdf).

Part 4: Management & Cleanup

Updating

cd /root/docker-deployment/stirling-pdf
docker compose pull
docker compose up -d

Cleaning Up Old Images

docker image prune -f

Complete Uninstall

To completely remove Stirling-PDF, its data, and images:

  1. Remove Proxy Host in Nginx Proxy Manager.
  2. Stop and Remove Data: 
    cd /root/docker-deployment/stirling-pdf
docker compose down -v
cd ..
rm -rf stirling-pdf
  3. Remove Image: 
    docker rmi stirlingtools/stirling-pdf:latest
