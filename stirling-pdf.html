What is Stirling-PDF?

Stirling-PDF allows you to perform almost any operation on PDF files directly from your browser. Stop uploading your private documents to random "Free Online PDF Merger" websites.

Features Include: Splitting, Merging, Rotating, converting PDFs to Images (and vice versa), Password protection, OCR (Optical Character Recognition), and much more.

Part 1: Installing with Docker

We will set up Stirling-PDF in its own dedicated directory.

Step 1: Create the Project Directory

# Create a directory for the service
mkdir /root/docker-deployment/stirling-pdf
cd /root/docker-deployment/stirling-pdf

Step 2: Create the `docker-compose.yml` File

nano docker-compose.yml

Paste the following configuration. We map the internal port 8080 to host port 8085 for testing.

services:
  stirling-pdf:
    image: frooodle/s-pdf:latest
    container_name: stirling-pdf
    restart: unless-stopped
    ports:
      # Maps host port 8085 to container port 8080
      - '8085:8080'
    volumes:
      # Optional: Persist OCR training data and configs
      - ./trainingData:/usr/share/tesseract-ocr/4.00/tessdata
      - ./extraConfigs:/configs
      - ./logs:/logs
    environment:
      # ENABLE SECURITY: Required to change admin settings
      - DOCKER_ENABLE_SECURITY=true
      - INSTALL_BOOK_AND_ADVANCED_HTML_OPS=false
      - LANGS=en_US
    networks:
      - nginx-proxy

# Connect to your existing shared network
networks:
  nginx-proxy:
    external: true

Step 3: Launch the Service

docker compose up -d

Default Credentials:
Username: admin
Password: stirling
(Please change these in the settings immediately)

Part 2: Securing with Nginx Proxy Manager

To access this securely via a domain name (e.g., `pdf.yourdomain.com`) and HTTPS.

  1. Point a Subdomain: In your domain registrar, create an 'A' record for your chosen subdomain pointing to your server's IP.
  2. Log in to Nginx Proxy Manager.
  3. Add a New Proxy Host:
    • Details Tab:
      • Domain Names: pdf.yourdomain.com
      • Scheme: http
      • Forward Hostname / IP: stirling-pdf (Matches the container name).
      • Forward Port: 8080 (The internal container port).
    • SSL Tab: Select "Request a new SSL Certificate" and turn on "Force SSL".
    • Save.

Your PDF tool is now ready to use at https://pdf.yourdomain.com.

Part 4: Management & Cleanup

Common tasks for maintaining, updating, or removing the service.

1. Updating to the Latest Version

Stirling-PDF gets frequent updates. To grab the latest features:

# Navigate to the directory
cd /root/docker-deployment/stirling-pdf

# Pull the latest image
docker compose pull

# Restart the service (this replaces the old container with the new one)
docker compose up -d

# (Optional) Clean up the old, unused image version to save space
docker image prune

2. "Hard Reset" (Wipe & Reinstall)

If the service is acting strange and you want to delete it and start fresh (keeping your config file):

cd /root/docker-deployment/stirling-pdf

# Stop and remove the container
docker compose down

# Start it up again fresh
docker compose up -d

3. Clean Up "Useless" Images

If you have updated multiple times, or uninstalled other services, you may have "dangling" images taking up space. This command removes them safely.

# Removes images that are not associated with any container
docker image prune

4. Complete Decommissioning (Uninstall)

To completely remove Stirling-PDF, including its data and the downloaded image:

  1. Remove the Proxy Host: In Nginx Proxy Manager, delete the host for `pdf.yourdomain.com`.
  2. Stop and Remove Data: 
    cd /root/docker-deployment/stirling-pdf

# Stop the service and remove volumes (-v)
docker compose down -v
  3. Delete the Project Folder: 
    cd ..
rm -rf stirling-pdf
  4. Remove the Docker Image: 
    # This frees up the disk space used by the installer
docker rmi frooodle/s-pdf:latest
